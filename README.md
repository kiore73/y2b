# Telegram-бот для автоматизации YouTube Shorts + TikTok

Этот проект объединяет скачивание видео из TikTok, наложение оверлея и автоматическую загрузку в YouTube Shorts по расписанию.

## Структура проекта
- `bot.py` — запуск Telegram-бота (принимает видео/ссылки).
- `scheduler.py` — запуск планировщика (загружает видео на YouTube раз в 3 часа).
- `core/` — логика БД, обработки видео (FFmpeg) и YouTube API.
- `utils/` — конфигурация и настройки.
- `data/` — хранилище видео, очереди и токенов.
- `assets/` — здесь должен лежать ваш `overlay.mov`.

## 1. Настройка окружения
1. Создайте файл `.env` (образец я создал) и заполните:
   - `BOT_TOKEN` — от @BotFather.
   - `ADMIN_ID` — ваш ID в Telegram.
   - `FFMPEG_PATH` — путь к ffmpeg.exe.
   - `OVERLAY_PATH` — путь к оверлею.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

## 2. Настройка YouTube
1. Положите `client_secrets.json` в корень проекта.
2. **Авторизация на VPS (без браузера)**:
   Поскольку на сервере нет браузера, стандартный вход не сработает. Рекомендуемый способ:
   - Запустите проект (или отдельный скрипт авторизации) на **локальном компьютере**.
   - Пройдите авторизацию в браузере.
   - В папке `data/tokens/` появятся файлы `.pickle` (например, `Channel_1_token.pickle`).
   - Просто скопируйте всю папку `data/tokens/` со своего компьютера на VPS в ту же директорию проекта.
   - После этого `scheduler.py` на сервере увидит токены и начнет загрузку автоматически.

## 3. Запуск
Для полноценной работы нужно запустить **два процесса** одновременно (на VPS используйте `screen`, `tmux` или `systemd`):

1. **Запуск бота**:
   ```bash
   python bot.py
   ```
2. **Запуск планировщика**:
   ```bash
   python scheduler.py
   ```

## Как пользоваться
1. Отправьте боту ссылку на TikTok или видеофайл.
2. Дождитесь наложения оверлея.
3. Введите название/описание или выберите стандартное.
4. Выберите канал.
5. Видео попадет в очередь. Раз в 3 часа `scheduler.py` будет проверять очередь и загружать по одному видео для каждого канала.

## Настройка на VPS (Linux)
Для работы 24/7 создайте две службы systemd (шаблоны в `README.md` предыдущей версии или используйте `pm2`).
